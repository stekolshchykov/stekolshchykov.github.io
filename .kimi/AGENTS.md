# Áras Living Website — Agent Instructions

## Project

- **Brand:** Áras Living
- **Domain:** arasliving.ie (primary)
- **Framework:** SvelteKit 2.x with `adapter-static`
- **Language:** TypeScript
- **Styling:** Tailwind CSS v4 (CSS-only config)
- **Package manager:** pnpm
- **Target host:** Vercel

## Conventions

- Use Svelte 5 runes (`$state`, `$derived`, `$effect`, `$props`).
- All components use `<script lang="ts">`.
- Tailwind v4 theme tokens are defined in `src/app.css` via `@theme`.
- Components live in `src/lib/components/{atoms,molecules,organisms}/`.
- Content lives in `src/lib/data/*.ts` as typed objects.
- Stores live in `src/lib/stores/*.svelte.ts`.
- Pages live in `src/routes/[[lang]]/...`.
- Use `trailingSlash: 'always'` and `prerender = true` globally.
- All user-facing strings are sourced from `src/lib/i18n/locales/en.json`.
- Images: use `@sveltejs/enhanced-img` for source images; static assets go to `static/`.
- GSAP ScrollTrigger: register once in root layout; use `gsap.context()` per component; revert on unmount.
- Form submits client-side to Formspree/Basin endpoint from `PUBLIC_FORMSPREE_ID`.
- Analytics scripts load only after cookie consent grant.

## Design v2 Notes

- The site follows a Poggenpohl +SEGMENTO-inspired premium direction: calm objectivity, material-first palette, architectural typography, image-led hierarchy.
- Accent is `#A68A64` (quiet brass/bronze); primary text is warm off-white `#F5F3EF`; hairlines use stone `#B8B4AD`.
- No rounded corners on images; no generic Lucide icons in service/why-us sections; no WhatsApp FAB.
- Buttons: `primary` (off-white fill on dark), `secondary` (1px text-primary outline), `outline-stone`, `ghost` (arrow text-link).
- Headings support a `reveal` prop for GSAP clip-mask reveals; `display` variant is h1-only.
- The hero section uses a static poster image (`static/images/hero-poster.jpg`) instead of video.
- Placeholder images are generated by `generate_images.py` (Pillow) and must be replaced with commissioned photography before launch.
- Brand voice is restrained and architectural; avoid superlatives and clichés. See historical content audits in `.kimi/output/reports/`.

## Build & Quality

- `pnpm dev` — start dev server
- `pnpm build` — generate sitemap and build static site
- `pnpm check` — Svelte + TypeScript type check
- `pnpm lint` — ESLint
- `pnpm format` — Prettier

Before finishing any task, run `pnpm check` and `pnpm build` and fix errors.
